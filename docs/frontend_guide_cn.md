# 前端开发与构建指南

本文档专为不熟悉前端生态系统的开发者编写，详细说明了如何在本机运行前端页面的“**开发模式**”以及如何最终“**编译打包部署**”。

---

## 1. 概念与背景
本项目的前端采用了目前最流行的 **Vue 3** 框架，并结合了极其快速的 **Vite** 构建工具。

由于前端页面（Vue + Vite）与后端服务（FastAPI）是分离的两个系统：
- **开发模式**：Vite 会默认在一个独立端口 (比如 `5173`) 启动一个实时的、支持热更新 (HMR) 的临时开发服务器。
- **后端服务**：FastAPI 提供数据 API，运行在 `9949` 端口。

为了能让前端发出的接口请求顺利抵达后端的 `9949` 端口，必须在 Vite 配置文件 `vite.config.js` 中配置一个 **代理 (Proxy)**。

> **提示**：目前已经在 `vite.config.js` 中写入了代理地址，当前端在开发环境请求 `/api/...` 时，会自动转发给 `http://127.0.0.1:9949`。

---

## 2. 本地开发模式
如果你需要对页面的 UI（颜色、按钮、排版等）进行修改和实时预览，请使用此模式。

**操作步骤**：
1. **启动后端服务**：打开一个终端，启动 FastAPI 服务：
   ```bash
   python llm_chat_server.py --host 0.0.0.0 --port 9949
   ```

2. **启动前端开发服务器**：打开另一个终端，进入 `frontend` 文件夹：
   ```bash
   cd frontend
   npm install
   npm run dev
   ```

3. **浏览器预览**：
   访问命令行里输出的地址（通常是 `http://localhost:5173/chat/`）。当你在编辑器中修改了代码并保存时，页面将瞬间自动更新。

---

## 3. 构建与部署上线
当你完成代码修改并准备上线时，需要对其进行构建打包。

**操作步骤**：
1. **停止前端开发服务器**：在终端里按下 `Ctrl + C`。
2. **执行构建命令**：
   ```bash
   npm run build
   ```
   将会生成一个 `dist` 文件夹，里面就是编译打包后的静态文件。
3. **验证部署**：
   将上面 `dist` 内的所有文件复制到 `static` 文件夹内。

   只需访问后端地址即可：
   ```text
   http://localhost:9949/chat/index.html
   ```

## 4. 多模型配置（本地保存）使用说明

前端已支持在「模型配置」抽屉中维护多个大模型配置档，这些配置保存在浏览器本地（`localStorage`），不会写入后端 `config.yaml`。

**你可以做的操作**：
1. **新建配置**：点击 `➕` 创建一个空白配置。
2. **复制配置**：点击 `📋` 基于当前配置复制一份，再修改参数。
3. **重命名配置**：点击 `✏️` 修改配置名称。
4. **删除配置**：点击 `🗑️` 删除当前本地配置（服务器默认配置不可删除）。
5. **切换配置**：通过下拉框切换不同配置快速测试不同模型/参数组合。

**与会话的关系**：
- 每个会话会记录当前配置名 `config_name`。
- 打开历史会话时，前端会自动尝试切换到当时对应的配置，保证上下文和模型设置一致。

### 4.1 命名输入框行为说明（重命名/复制）
- 当前版本使用浏览器系统 `prompt` 作为输入框，而不是自定义弹窗组件。
- 重命名默认值：`${oldName}-New`；复制默认值：`${oldName}-Copy`。
- 由于 `prompt` 是浏览器/宿主实现，默认值是否展示、光标位置、是否自动选中文本等行为，可能在不同运行环境（浏览器、WebView、IDE 内嵌预览）存在差异。
- 若你需要“始终预填 + 自动全选 + 可控样式”的一致体验，建议改为前端自定义弹窗实现。

### 4.2 配置存储与恢复（实现细节）
- 配置数组存储键：`llm_chat_configs`。
- 当前选中索引键：`llm_chat_config_index`。
- 页面初始化时会从后端拉取最新配置，并强制覆盖 `configList[0]`（Server 配置），确保默认配置与后端实际值一致。
- 若本地缓存 JSON 损坏或结构非法，会自动清理缓存并回退到仅保留 Server 配置，避免页面初始化失败。
