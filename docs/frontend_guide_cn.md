# 前端开发与构建指南

本文档专为不熟悉前端生态系统的开发者编写，详细说明了如何在本机运行前端页面的“**开发模式**”以及如何最终“**编译打包部署**”。

---

## 1. 概念与背景
本项目的前端采用了目前最流行的 **Vue 3** 框架，并结合了极其快速的 **Vite** 构建工具。

由于前端页面（Vue + Vite）与后端服务（FastAPI）是分离的两个系统：
- **开发模式**：Vite 会默认在一个独立端口 (比如 `5173`) 启动一个实时的、支持热更新 (HMR) 的临时开发服务器。
- **后端服务**：FastAPI 提供数据 API，运行在 `9949` 端口。

为了能让前端发出的接口请求顺利抵达后端的 `9949` 端口，必须在 Vite 配置文件 `vite.config.js` 中配置一个 **代理 (Proxy)**。

> **提示**：我已经帮你把代理写入了 `vite.config.js`，当前端在开发环境请求 `/api/...` 时，会自动转发给 `http://127.0.0.1:9949`。

---

## 2. 本地开发模式
如果你需要对页面的 UI（颜色、按钮、排版等）进行修改和实时预览，请使用此模式。

**操作步骤**：
1. **启动后端服务**：打开一个终端，启动 FastAPI 服务：
   ```bash
   python llm_chat_server.py --host 0.0.0.0 --port 9949
   ```

2. **启动前端开发服务器**：打开另一个终端，进入 `frontend` 文件夹：
   ```bash
   cd frontend
   npm run dev
   ```

3. **浏览器预览**：
   访问命令行里输出的地址（通常是 `http://localhost:5173/chat/`）。当你在编辑器中修改了代码并保存时，页面将瞬间自动更新。

---

## 3. 构建与部署上线
当你完成代码修改并准备上线时，需要对其进行构建打包。

**操作步骤**：
1. **停止前端开发服务器**：在终端里按下 `Ctrl + C`。
2. **执行构建命令**：
   ```bash
   npm run build
   ```
3. **验证部署**：
   本项目已经在后端目录创建了名为 `static` 的文件夹链接，指向了 `frontend/dist/`。构建完成后，最新代码会直接生效。

   只需访问后端地址即可：
   ```text
   http://127.0.0.1:9949/chat/
   ```
